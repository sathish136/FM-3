Project Drawing Module – Complete Workflow
1. Drawing Upload
The first user uploads project drawings in bulk.
2. Drawing Review & Correction
Another user reviews the uploaded drawings.
If all drawings are correct:
The reviewer submits them for approval.
If corrections are required:
The reviewer marks corrections using the markup tool.
The reviewer can add comments, notes, or instructions on the drawing.
3. Correction Notification
After corrections are marked:
A notification is sent to the uploader.
Notification message:
“Reviewer has requested corrections for the drawing.”
4. Corrected Drawing Re-Upload
The uploader corrects the drawings and uploads the updated files again.
Once updated:
A notification is sent to the reviewer for rechecking.
Revision Management (REV System)
5. Revision Control
After approval, if any additional correction is required:
The drawing revision number should increase automatically.
Example:
First approved drawing → REV-0
After first correction → REV-1
Next correction → REV-2
6. Revision Workflow
Every revision follows the same workflow:
Upload revised drawing
Reviewer checks drawing
Corrections marked if required
Re-upload corrected drawing
Final approval process
Complete revision history must be maintained.
Final Approval Workflow
7. Design HOD Approval
Once the reviewer submits the drawing:
The Design HOD reviews the final drawing.
If approved:
Notification sent to the Project Manager.
Project Manager Module (Separate Module)
8. Final Approved Drawing Module

After final approval:

Approved drawings move to a separate module accessible by the Project Manager.
Project Manager Features

The Project Manager can:

View all approved drawings
Download drawings
Share drawings
Give access permissions to:
Workshop Team
Site Engineers
Production Team
Workshop & Site Engineer Module (Separate Module)
9. Site / Workshop Follow-Up
Workshop and Site Engineers can:
View approved drawings
Track execution progress
Report issues found during production or site work
10. Issue Reporting System

If any issue or correction is found during execution:

Users can:
Mark issues directly on drawings
Add comments and reports
Upload supporting images/documents
Example Reports
Dimension mismatch
Fabrication issue
Site installation issue
Material conflict
Missing details
Separate Issue Correction Workflow
Reported issues should go through a separate correction workflow.
Notifications should be sent to:
Design Team

Project Manager

Revision tracking process
